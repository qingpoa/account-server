## 1. 项目总览
### 1.1 项目背景
本系统为个人财务收支管理工具，支持用户在线记账、多维度数据统计分析、数据导出、AI智能记账、AI财务分析、分类预算管理等功能，满足个人日常财务管理需求。

### 1.2 项目目标
- 实现个人收支数据的全生命周期管理（增删改查）
- 提供可视化财务分析能力，帮助用户掌握收支结构
- 支持分类预算设置与超支提醒，帮助用户控制消费
- 支持AI自然语言快速记账，降低记账门槛
- 支持AI智能财务分析，提供专业的消费优化建议
- 系统架构模块化，可扩展、易维护，符合软件工程课程设计规范

### 1.3 需求分析
#### 1.3.1 功能性需求
- **用户管理：** 支持用户注册、登录、信息修改、密码重置等基础身份管理功能，确保用户数据隔离与安全
- **账单管理：** 支持收支账单的新增、编辑、删除、多条件筛选查询，分类为系统预设，用户直接选择即可
- **预算管理：** 支持用户为每个预设收支分类设置月度/季度/年度预算，支持预算进度查看、超支自动提醒
- **统计分析：** 支持多维度财务数据可视化展示，包括收支总览、分类占比、月度趋势、预算完成度等统计维度，直观展示用户财务状况
- **数据导出：** 支持按时间范围、收支类型、分类等条件筛选导出账单数据为Excel格式，便于用户存档与二次分析
- **AI增强功能：** 自然语言自动记账、AI财务分析报告生成、大模型配置管理

#### 1.3.2 非功能性需求
| 维度 | 要求 |
|------|------|
| 性能 | 接口平均响应时间 < 200ms，AI接口响应时间 < 3s，单页加载时间 < 1s |
| 可靠性 | 数据备份机制，无数据丢失风险，系统年可用率 > 99.5% |
| 易用性 | 操作流程符合大众使用习惯，新用户上手时间 < 5分钟，AI记账识别准确率 > 90% |
| 兼容性 | 支持Chrome、Edge、Firefox主流浏览器，适配PC端1920*1080及以上分辨率 |
| 安全性 | 密码加盐加密存储、接口token鉴权、用户数据完全隔离、XSS/CSRF防护、API Key加密存储 |
| 可扩展性 | 系统采用模块化架构，支持后续快速扩展多端适配、账单自动导入、共享账本等功能 |

### 1.4 可行性分析

## 3. 数据库设计
### 3.1 表结构总览
共6张表，所有表均支持逻辑删除，用户数据通过`user_id`完全隔离：
`user` ↔ 1:N ↔ `bill` | `system_category` ↔ 1:N ↔ `bill` | `user` ↔ 1:N ↔ `budget_config` | `user` ↔ 1:1 ↔ `ai_config` | `user` ↔ 1:N ↔ `ai_report`

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS pfms DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_general_ci;
USE pfms;

-- 1. 用户表 user
CREATE TABLE `user` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '用户唯一ID',
  `username` varchar(32) NOT NULL COMMENT '用户名',
  `password` varchar(255) NOT NULL COMMENT '加密后密码',
  `nickname` varchar(32) DEFAULT NULL COMMENT '昵称',

  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint DEFAULT '0' COMMENT '逻辑删除标记 0=正常 1=删除',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)

-- 2. 系统预设分类表 system_category（初始化后固定数据）

  `icon` varchar(64) DEFAULT NULL COMMENT '分类图标标识',
  `sort` int DEFAULT '0' COMMENT '排序权重 越大越靠前',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_type` (`type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统预设分类表';

-- 3. 预算配置表 budget_config
CREATE TABLE `budget_config` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '预算唯一ID',

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='预算配置表';

-- 4. 账单表 bill

  `user_id` bigint NOT NULL COMMENT '所属用户ID',

  `amount` decimal(10,2) NOT NULL COMMENT '金额 单位元',
  `type` tinyint NOT NULL COMMENT '类型 1=收入 2=支出',
  `remark` varchar(255) DEFAULT NULL COMMENT '备注信息',
  `record_time` datetime NOT NULL COMMENT '收支发生时间',

  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` tinyint DEFAULT '0' COMMENT '逻辑删除标记',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_record_time` (`record_time`),
  KEY `idx_type` (`type`)

-- 5. AI配置表 ai_config
```

### 3.4 数据库优化建议
- 所有关联`user_id`的查询字段都已加索引，保证多用户场景下的查询性能
- 账单表按`record_time`范围查询较多，已加索引优化统计查询速度
- 敏感字段（密码、API Key）必须加密存储，禁止明文入库
- 建议配置MyBatis-Plus全局逻辑删除，避免手动写`is_deleted=0`条件
- 生产环境建议配置数据库定时备份策略，避免数据丢失

   - 全局异常处理器，映射通用错误码
   - 跨域配置
   - 分页插件配置
3. **鉴权组件**：
   - JWT Token生成、校验工具类
   - 全局拦截器：拦截除登录/注册外所有接口，校验Token有效性，解析用户ID存入上下文
   - 密码加密工具：使用BCrypt对密码加盐加密存储
4. **数据隔离组件**：所有涉及用户数据的查询自动拼接`user_id = 当前登录用户ID`条件，避免越权访问

### 4.3 业务模块开发优先级
| 序号 | 模块 | 核心功能点 | 对应接口数 |
|------|------|------------|------------|
| 1 | 用户模块 | 注册、登录、密码修改、个人信息修改、超支提醒开关配置 | 5个 |
| 2 | 系统分类模块 | 系统预设分类列表查询（数据初始化后固定，用户不可修改） | 1个 |
| 3 | 账单模块 | 账单CRUD、多条件筛选查询、新增账单时自动校验是否超支并返回提醒 | 5个 |
| 4 | 预算模块 | 预算配置CRUD、预算进度自动计算（已使用/剩余/完成率）、超支判断逻辑 | 4个 |
| 5 | 统计模块 | 多维度数据聚合计算（收支总览、分类占比、月度趋势、预算执行） | 4个 |
| 6 | 导出模块 | Excel文件生成、文件流下载返回 | 1个 |
| 7 | AI模块 | 大模型API对接封装、提示词模板、AI记账语义解析、财务报告生成、API Key加密存储 | 7个 |

## 5. 接口文档
### 5.1 通用规范
- **请求根路径：** `/api/v1`
- **鉴权规则：** 所有接口（登录/注册除外）请求头需携带`Authorization: Bearer {token}`
- **统一响应格式：**
```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```
- **通用错误码：**
  | 错误码 | 说明 |
  |--------|------|
  | 200 | 请求成功 |
  | 400 | 参数错误 |
  | 401 | 未登录/登录过期 |
  | 402 | API Key无效/调用额度不足 |
  | 403 | 无权限访问 |
  | 404 | 资源不存在 |
  | 500 | 服务器内部错误 |
  | 503 | 大模型服务调用失败 |
